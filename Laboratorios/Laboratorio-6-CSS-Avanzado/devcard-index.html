Reflexión

1. ¿Qué ventajas ofrecen las variables CSS sobre los valores hardcodeados?

Permiten reutilizar valores en todo el CSS, facilitan cambios (como colores o espaciado) rapidos desde un solo lugar y hacen el código más consistente.

2. ¿En qué casos conviene declarar variables CSS fuera del selector :root?

Cuando quieres que las variables solo apliquen a un componente o contexto específico, por ejemplo dentro de una clase como .card o .dark.

3. ¿Cómo se pueden utilizar las variables CSS para implementar un sistema de temas (como modo oscuro)?

Definiendo variables en :root y luego sobrescribiéndolas en otra clase (como body.dark). Al cambiar la clase, cambian automáticamente todos los estilos que usan esas variables.

4. ¿Qué sucede cuando intentas usar una variable CSS que no ha sido declarada?

El valor no se aplica y la propiedad se ignora.

5. ¿Cuál es la diferencia entre los enfoques mobile-first y desktop-first?

Mobile-first empieza diseñando para pantallas pequeñas (como smartphones) y luego escala con media queries, desktop-first hace lo contrario, empieza en grande (table y desktop) y adapta hacia pantallas pequeñas.

6. ¿Qué breakpoints son los más comunes y por qué?

Aproximadamente 768px (tablet) y 1024px (desktop), porque corresponden a resoluciones comunes y permiten adaptar bien el diseño entre móvil, tablet y desktop.

7. ¿Cómo afecta el viewport meta tag al diseño responsivo?

Permite que el navegador ajuste el ancho de la página al tamaño del dispositivo, evitando que el diseño se vea como “zoom out” en móviles.

8. ¿Qué técnicas además de media queries existen para crear diseños responsivos?

Flexbox, CSS Grid, unidades relativas a la pantalla(%, rem, vw), imagenes responsivas.

9. ¿Cómo puedes identificar qué regla CSS está siendo aplicada cuando hay múltiples selectores?

Usando DevTools en el navegador, revisando la pestaña “Styles”, donde se ve qué reglas se aplican y cuáles son sobrescritas.

10. ¿Qué información proporciona la pestaña Computed en DevTools?

Muestra los valores finales que el navegador aplica a cada propiedad.

11. ¿Cómo puedes simular diferentes dispositivos sin tener acceso físico a ellos?

Usando las herramientas de desarrollador del navegador (Device Toolbar) para cambiar resoluciones y emular dispositivos.

12. ¿En qué situaciones es preferible usar Bootstrap sobre Tailwind CSS y viceversa?

Bootstrap es ideal para desarrollo rápido con componentes predefinidos, mientras que Tailwind CSS es más adecuado para proyectos donde se requiere un diseño altamente personalizado y control total sobre los estilos.

13. ¿Qué significa que Tailwind sea "utility-first"?

Que usa clases pequeñas y de un solo proposito (como p-4, text-center) para construir estilos directamente en el HTML en lugar de escribir en un CSS .

14. ¿Cómo afecta el uso de un framework CSS al tamaño final del bundle?

Puede aumentarlo si no se optimiza, ya que incluye muchas clases o componentes, aunque herramientas como PurgeCSS pueden recuir el tamaño final.

15. ¿Es posible combinar múltiples frameworks CSS en un mismo proyecto? ¿Por qué sí o por qué no?

Sí es posible combinar múltiples frameworks CSS en un mismo proyecto, pero no es recomendable, ya que puede generar conflictos entre estilos, inconsistencias en el diseño y un aumento innecesario en el tamaño del proyecto.
