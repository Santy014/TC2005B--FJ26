Elementos que se utilizan en desarrollo Web:

¿Por que es una buena practica usar JavaScript para checar que sean validos los inputs de las formas antes de enviar los datos al servidor?

Porque mejora la experiencia del usuario: le avisas al instante si falta un dato, si el correo está mal escrito o si una contraseña no cumple reglas, sin esperar la respuesta del servidor. También reduce peticiones innecesarias al backend.

¿Como puedes saltarte la seguridad de validaciones hechas con JavaScript?

Se puede saltar fácilmente porque JavaScript corre en el navegador del usuario. Por ejemplo, alguien podría desactivar JavaScript, modificar el HTML desde las herramientas de desarrollador, cambiar el código de JavaScript, o enviar la petición directamente con herramientas como Postman.

Si te puedes saltar la seguridad de las validaciones de JavaScript, entonces ¿por que la primera pregunta dice que es una buena practica?

Porque validar en JavaScript no es seguro por si mismo, es una capa de comodidad para el usuario y una prevencion temprana. La seguridad real debe estar del lado del servidor, que son aquellas que van a dar seguridad.

Navegación entre paginas utilizando la etiqueta <a>:

La etiqueta <a> sirve para navegar entre páginas internas o externas.

Diferencias entre navegación con formularios (submit) y botones con JavaScript:

Un <form> con submit se usa cuando se quieren enviar datos, Un <button> con JavaScript se usa cuando se quiere controlar la acción manualmente. La principal Diferencias es que el <submit> pertenece al flujo normal de formularios HTML, mientras que el botón con JavaScript depende del código que nosotros escribamos.

Rutas absolutas y relativas:

Una ruta absoluta empieza desde la raíz del sitio, busca desde el dominio principal al desplegar la pagina, una ruta relativa depende de la ubicación del archivo actual en el que se encuentra, si el archivo está en otra carpeta, la ruta puede cambiar y romperse.
